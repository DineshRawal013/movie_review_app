# DEF-003 — Admin review moderation endpoint mismatch vs api-spec.yaml (PATCH vs POST)

## Requirement Violated

- api-spec.yaml: Defines admin hide/restore as `PATCH /admin/reviews/{reviewId}` with body `{ "action": "hide" | "restore" }`.
- TC-ADMIN-003 (step 1): `PATCH /api/admin/reviews/{reviewId}` with `{ "action": "hide" }`.
- TC-ADMIN-004 (step 1): `PATCH /api/admin/reviews/{reviewId}` with `{ "action": "restore" }`.

---

## Description

In `backend/src/admin/admin.controller.ts`, the hide and restore actions are implemented as two separate `POST` endpoints instead of a single `PATCH` endpoint with an action body parameter:

```typescript
@Post('reviews/:id/hide')
hideReview(@Param('id') id: string, @CurrentUser() user: any) { ... }

@Post('reviews/:id/restore')
restoreReview(@Param('id') id: string, @CurrentUser() user: any) { ... }
```

The approved api-spec.yaml specifies:
```
PATCH /admin/reviews/{reviewId}
Body: { "action": "hide" | "restore" }
```

The implementation uses non-standard action-verb URLs (`/reviews/:id/hide`, `/reviews/:id/restore`) with POST method, diverging from the REST-style PATCH contract in the api-spec.

---

## Steps to Reproduce

1. Send `PATCH /api/admin/reviews/{reviewId}` with body `{ "action": "hide" }` and admin auth.
2. Observe: HTTP 404 (no route registered for this method+path).
3. Send `POST /api/admin/reviews/{reviewId}/hide` — this works.

---

## Expected Result

- `PATCH /api/admin/reviews/{reviewId}` with `{ "action": "hide" }` returns HTTP 200.
- `PATCH /api/admin/reviews/{reviewId}` with `{ "action": "restore" }` returns HTTP 200.

---

## Actual Result

- `PATCH /api/admin/reviews/{reviewId}` returns 404.
- Functional equivalent exists at `POST /admin/reviews/:id/hide` and `POST /admin/reviews/:id/restore`.

---

## Fix Guidance

Two options (choose one; update api-spec.yaml if Option B selected):

**Option A — Fix implementation to match api-spec (recommended):**
Replace the two `@Post` handlers with a single `@Patch` handler:
```typescript
@Patch('reviews/:id')
moderateReview(
  @Param('id') id: string,
  @Body('action') action: 'hide' | 'restore',
  @CurrentUser() user: any,
) {
  if (action === 'hide') return this.adminService.hideReview(id, user.id);
  if (action === 'restore') return this.adminService.restoreReview(id, user.id);
  throw new BadRequestException('Invalid action');
}
```

**Option B — Raise a Change Request to update api-spec.yaml to match implementation.**
