**Severity:** P2 — Major

## Requirement Violated

- FR-060 — User profile page at /profile (own) and /users/{userId} (public).
- FR-061 — Profile shows: display name, avatar, member-since, reviews list with movie links.
- TC-AUTH-009 (step 1): `GET /api/users/{aliceUserId}` without authentication — verify HTTP 200.
- TC-AUTH-010 (steps 1–3): `GET /api/users/{aliceUserId}` returns `displayName`, `avatarUrl`, `createdAt`, `reviews` array.
- api-spec.yaml: `GET /users/{userId}` returns `UserPublic` schema.

---

## Description

In `backend/src/users/users.controller.ts`, only `GET /users/me`, `PATCH /users/me`, and `DELETE /users/me` are defined. There is no `GET /users/:id` (public profile) endpoint:

```typescript
@Controller('users')
export class UsersController {
  @Get('me')           // GET /users/me — own profile (authenticated)
  getMe(...)

  @Patch('me')         // PATCH /users/me — update own profile
  updateMe(...)

  @Delete('me')        // DELETE /users/me — delete own account
  async deleteMe(...)
  // <-- Missing: @Get(':id') for public profile
}
```

The api-spec.yaml defines `GET /users/{userId}` returning a `UserPublic` schema (id, displayName, avatarUrl, createdAt, reviews). The frontend `user-profile.tsx` component uses `usersApi.me()` which calls `GET /users/me`, but there is no route to serve a public profile for another user.

Additionally, `UsersService.getProfile()` selects `email` in its response — which should NOT be returned in the public-facing `UserPublic` schema per the design (email is internal).

---

## Steps to Reproduce

1. Send GET `/api/users/{anyUserId}` (non-"me").
2. Observe: HTTP 404 (no route registered).

---

## Expected Result

- `GET /api/users/{userId}` returns HTTP 200 with `UserPublic` schema (no email field).
- Accessible by unauthenticated and authenticated users.

---

## Actual Result

- `GET /api/users/{userId}` returns HTTP 404.

---

## Fix Guidance

In `users.controller.ts`, add a public endpoint:
```typescript
@Public()
@Get(':id')
getPublicProfile(@Param('id') id: string) {
  return this.usersService.getPublicProfile(id);
}
```

In `users.service.ts`, add `getPublicProfile()` method that selects only public fields (no email):
```typescript
async getPublicProfile(id: string) {
  const user = await this.prisma.user.findUnique({
    where: { id },
    select: {
      id: true,
      displayName: true,
      avatarUrl: true,
      createdAt: true,
      reviews: {
        where: { deletedAt: null, isHidden: false },
        orderBy: { createdAt: 'desc' },
        take: 20,
        select: { id: true, body: true, createdAt: true, movie: { select: { id: true, tmdbId: true, title: true } } },
      },
    },
  });
  if (!user) throw new NotFoundException('User not found');
  return user;
}
```
