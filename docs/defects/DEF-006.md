# DEF-006 — Movie detail page missing runtime, director, cast fields (FR-021)

**Severity:** P2 — Major
**Status:** FIXED
**Raised by:** qa-engineer
**Date Raised:** 2026-05-26

---

## Requirement Violated

- FR-021 — Detail page shows title, poster, backdrop, release date, runtime, genres, synopsis, **director, cast**.
- TC-MOVIES-015 (steps 8–12): Verify runtime displayed, director name visible, cast member name visible.
- api-spec.yaml: `MovieDetail` schema includes `runtime`, `director`, `cast` array fields.

---

## Description

The `frontend/components/movies/movie-detail.tsx` component renders:
- title (H1)
- releaseDate
- genres (chips)
- overview (synopsis)
- StarRating widget (avgRating, ratingCount)

It does NOT render:
- `runtime` (e.g., "139 min")
- `director` name
- `cast` array

These are defined in FR-021 (P0) as required fields for the movie detail page.

Additionally, `backend/src/movies/movies.service.ts` `syncMovieFromTmdb()` stores only `title`, `overview`, `releaseDate`, `posterUrl`, `backdropUrl` to the database. The `runtime`, `director`, and `cast` fields from the TMDB response (which includes `credits` via `append_to_response: 'credits,videos'` in `tmdb.service.ts`) are not persisted or mapped.

The `trailerUrl` (FR-027) is also not being extracted from the TMDB `videos` response and stored, though `tmdb.service.ts` does request it via `append_to_response`.

---

## Steps to Reproduce

1. Navigate to any movie detail page (e.g., `/movies/550`).
2. Observe: No runtime, no director name, no cast list displayed.

---

## Expected Result

Movie detail page displays (per FR-021):
- Runtime: "139 min"
- Director: "David Fincher"
- Cast: "Brad Pitt, Edward Norton, Helena Bonham Carter" (at minimum first 3)

---

## Actual Result

Runtime, director, and cast fields are absent from both the backend API response and the frontend render.

---

## Fix Guidance

**Backend fix (`movies.service.ts` — `syncMovieFromTmdb()`):**
```typescript
const runtime = tmdbData.runtime ?? null;
const director = tmdbData.credits?.crew?.find((c: any) => c.job === 'Director')?.name ?? null;
const cast = tmdbData.credits?.cast?.slice(0, 10).map((c: any) => c.name) ?? [];
const trailerUrl = tmdbData.videos?.results?.find(
  (v: any) => v.type === 'Trailer' && v.site === 'YouTube'
)?.key
  ? `https://www.youtube.com/watch?v=${tmdbData.videos.results.find((v: any) => v.type === 'Trailer' && v.site === 'YouTube').key}`
  : null;
```
Add these fields to the Prisma `movie.upsert` `create` and `update` objects. Verify Prisma schema has `runtime Int?`, `director String?`, `cast String[]`, `trailer_url String?`.

**Frontend fix (`movie-detail.tsx`):**
Add display elements for runtime, director, and cast in the detail card JSX.
